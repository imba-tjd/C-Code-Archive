可实用的数据结构的基础就是造一个可自动扩展且可随机访问的线性表，否则啥都造不出来。

如果用双向链表，无法随机访问就不说了。如果没有指向尾部的指针，那append都无法忍受了，只有insert0可用，唯一可以造栈；如果有，那头结点就必不可能和普通结点一样。

而且无论用哪种，动态内存分配都必不可少。

泛型，普通的typedef不能同时使用不同类型的容器。若是全部实现在头文件中，include前typedef一下，类型倒是没问题，但函数会冲突，而且函数名中无法使用##解决

错误处理，就拿find来说，找不到怎么办？返回值肯定要是容器泛型类型，不可能每个可能出问题的都返回error_no，那根本无法好好使用。
